# ex2の課題

## 課題の概要

本課題では，SoccerTrack Challenge 2025 の動画に対して Deep-EIoU による選手追跡を行い，得られたトラッキング結果を可視化・分析する．
併せて，速度や加速度などの特徴量を用いてトラッキング結果を確認し，TrackEval による評価指標の意味を理解することを目的とする．

対象ノートブック：

- `5.2_Soccer_Track_Challenge_2025.ipynb`

## 課題

1. Deep-EIoU の出力を用いて，選手の軌跡を可視化する
2. 速度または加速度を計算し，トラッキング結果の特徴や異常な動きを分析する
3. Deep-EIoU の設定や後処理を変更し，TrackEval の評価値を比較する（発展）

## 課題の進め方

1. ノートブックの処理を確認する
   - SoccerTrack Challenge 2025 の動画 `117093.mp4` をダウンロードする
   - 先頭20秒を `117093_trimmed.mp4` として切り出す
   - Deep-EIoU を実行し，MOTChallenge形式のトラッキング結果を出力する
   - TrackEval を用いて，HOTA，CLEAR，Identity などの指標で評価する

2. 課題1：選手軌跡の可視化
   - Deep-EIoU の出力txtを読み込む
   - 各行の `frame`, `id`, `x`, `y`, `w`, `h` を使い，バウンディングボックスの中心座標を計算する
   - 代表的な数名の `track_id` について，中心座標の軌跡を2D平面上に描画する
   - 軌跡の途切れ，急なジャンプ，IDスイッチらしき箇所がないか確認する

   Deep-EIoU の予測結果は，通常以下の場所に出力される：

   ```text
   /content/Deep-EIoU/Deep-EIoU/YOLOX_outputs/yolox_x_ch_sportsmot/track_vis/*.txt
   ```

   中心座標は以下のように計算できる：

   ```python
   cx = x + w / 2
   cy = y + h / 2
   ```

3. 課題2：速度・加速度を用いた分析
   - 各 `track_id` について，フレーム間の中心座標の差から速度を計算する
   - 必要に応じて，速度の変化から加速度を計算する
   - 異常に大きな速度や加速度が出ている箇所を探す
   - その箇所が，誤検出，IDスイッチ，カメラ移動，選手の実際の動きのどれに近いかを考察する

   速度と加速度は以下のように計算できる：

   ```python
   vx = cx_t - cx_t_minus_1
   vy = cy_t - cy_t_minus_1
   speed = (vx**2 + vy**2)**0.5
   acceleration = speed_t - speed_t_minus_1
   ```

   余裕があれば，異常点を除外または補正した結果を作成し，TrackEvalで評価値を比較する．

4. 課題3：トラッキング設定の比較（発展）
   - Deep-EIoU の元設定でトラッキングを実行する
   - パラメータを変更して，再度トラッキングを実行する
   - TrackEval を用いて，変更前後の HOTA，CLEAR，Identity を比較する
   - どの設定が良かったか，軌跡の可視化結果と合わせて説明する

   比較に使いやすいパラメータの例：

   - `track_high_thresh`
   - `track_low_thresh`
   - `new_track_thresh`
   - `track_buffer`
   - `match_thresh`
   - `aspect_ratio_thresh`
   - `min_box_area`
   - `nms_thres`
   - `--with-reid`
   - `proximity_thresh`
   - `appearance_thresh`

5. 発表に向けて整理する
   - 可視化した軌跡から分かる動きの特徴を説明する
   - 異常な動きがあった場合，その原因を考察する
   - 速度・加速度を追加することで何が分かるようになったか説明する
   - パラメータ変更や後処理によって評価値がどう変化したか説明する

## レポートに含める内容

1. 課題1：可視化・理解
   - 可視化した選手ID
   - 軌跡の図
   - 動きの特徴
   - 異常な動きや軌跡の途切れがあったか

2. 課題2：特徴量の改変
   - 追加した特徴量（速度または加速度）
   - 計算方法
   - 特徴量から分かったこと
   - 可能であれば，補正前後または除外前後の評価値比較

3. 課題3：モデル・設定の比較（発展）
   - 比較した設定
   - HOTA，CLEAR，Identity などの評価値
   - どちらが良かったか
   - なぜその結果になったと考えられるか

## ヒント

- Pythonを使う場合
  - データ処理：`numpy`, `pandas`
  - 描画：`matplotlib`
  - 動画処理：`ffmpeg`
  - 評価：`TrackEval`
- MOTChallenge形式の予測ファイルは，おおよそ以下の形式になっている

  ```text
  frame, id, x, y, w, h, score, class
  ```

- このノートブックでは，20秒，25fps，合計500フレームを評価対象としている
- フレーム番号は1から始まる必要がある
- サッカー映像ではカメラが動くため，画像座標上の速度がそのままピッチ上の実速度を表すとは限らない

## 注意

- Colabで実行する場合は，ランタイムをGPUに設定すること
- ノートブック内のパスは `/content/` を前提としているため，ローカル実行する場合はパスを修正すること
- トラッキング結果のtxtファイルや動画は，設定を変えて実行する前に保存しておくこと
- グラフの軸ラベル，凡例，対象とした `track_id` を明記すること
- 評価値だけでなく，可視化結果も使って考察すること
- 発展課題までできなくても，まずは軌跡の可視化と速度・加速度の分析を完成させること
