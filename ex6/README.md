# ex6の課題

## 課題の概要

本課題では，Soccer Trajectory Prediction Challenge 2025 のデータを用いて，サッカーにおける選手とボールの将来軌道を予測する．
RNNモデルを用いたベースラインを実行し，入力軌道，Ground Truth，予測結果を比較することで，時系列データとしてのサッカーの動きを理解することを目的とする．

対象ノートブック：

- `5.5_Soccer Trajectory Prediction Challenge 2025.ipynb`

## 課題

1. 入力軌道，Ground Truth，予測軌道を可視化する
2. 複数ファイルに対して，ボール・左チーム・右チームの誤差を分けて評価する
3. 入力フレーム数や速度などの特徴量を変更し，予測結果の変化を分析する
4. モデルや条件を変更し，元のRNNモデルと比較する（発展）

## 課題の進め方

1. ノートブックの処理を確認する
   - 必要なパッケージをインストールする
   - `STP-challenge-2025` のGitHubリポジトリをクローンする
   - RNNモデルを学習する
   - challenge data に対して予測を実行する
   - 予測結果を評価し，入力・GT・予測をピッチ上に可視化する

   ノートブックでは，以下のようにRNNモデルを実行している：

   ```bash
   python main.py --n_epoch 10 --model RNN --burn_in 30 -t_step 60 --Sanity
   ```

   この設定では，最初の30フレームを入力として使い，60フレーム目までの軌道を扱う．
   つまり，過去30フレームから将来30フレームを予測する設定になっている．

2. 課題1：軌道の可視化・理解
   - `./test_old/input/01.csv` など，少なくとも3ファイルを入力データとして読み込む
   - 対応する `./test_old/gt/*.csv` をGround Truthとして読み込む
   - 対応する `./results/test/submission/*.csv` を予測結果として読み込む
   - ピッチ上に，入力軌道，GT軌道，予測軌道を重ねて描画する
   - ボール，左チームの選手，右チームの選手を区別して表示する
   - 予測誤差が最大になるフレームまたは選手を図中または表で示す

   ノートブックでは，以下のような列を使っている：

   ```text
   b_x, b_y
   l*_x, l*_y
   r*_x, r*_y
   ```

   ここで，`b` はボール，`l` は左チームの選手，`r` は右チームの選手を表す．

   レポートでは，以下を説明する：

   - ボールの予測軌道はGTに近いか
   - 選手の予測軌道はGTに近いか
   - ボール，左チーム，右チームの平均誤差はそれぞれどの程度か
   - 予測が大きく外れている選手や場面はあるか
   - 入力軌道の動きから，予測しやすい場面・難しい場面はどこか

3. 課題2：特徴量・入力条件の変更
   - 次のうち2つ以上を選び，元の設定と比較する
     - 入力フレーム数 `--burn_in` を変更する
     - 予測対象フレーム数 `-t_step` を変更する
     - 速度を特徴量として追加する
     - 加速度を特徴量として追加する
     - ボールのみ，または特定選手のみを詳しく分析する
     - 予測軌道に対して，移動平均などの単純な後処理を加える

   速度は，隣接フレームの座標差から計算できる：

   ```python
   vx = x_t - x_t_minus_1
   vy = y_t - y_t_minus_1
   speed = (vx**2 + vy**2)**0.5
   ```

   加速度は，速度の時間変化として計算できる：

   ```python
   acceleration = speed_t - speed_t_minus_1
   ```

   入力フレーム数を変更する場合の例：

   ```bash
   python main.py --n_epoch 10 --model RNN --burn_in 20 -t_step 60 --Sanity
   python main.py --n_epoch 10 --model RNN --burn_in 40 -t_step 60 --Sanity
   ```

   レポートでは，以下を説明する：

   - どの条件を変更したか
   - 変更した条件を表にまとめる
   - 予測軌道は元の設定と比べてどう変化したか
   - 評価値は改善したか，悪化したか
   - 改善・悪化を，ボール・左チーム・右チームに分けて説明できるか
   - なぜそのような変化が起きたと考えられるか

4. 課題3：モデル・条件の比較（発展）
   - 元のRNNモデルと，自分で変更したモデルまたは条件を比較する
   - 可能であれば，複数の設定で学習・評価を行う
   - `example/evaluation.py` を用いて，元の予測結果と変更後の予測結果を比較する
   - 少なくとも1つの「改善しなかった設定」も取り上げ，なぜ改善しなかったかを考察する

   比較例：

   - RNNの元設定 vs 入力フレーム数を変更した設定
   - RNNの元設定 vs 予測フレーム数を変更した設定
   - RNNの元設定 vs 速度を考慮した後処理
   - RNNの元設定 vs 学習エポック数を増やした設定
   - Sanity用の少数データ学習 vs 1000試合データを用いた学習済みモデル

   ノートブックでは，1000ファイルで学習した重みを利用するオプションも示されている：

   ```text
   weights/RNN_robocup2D/512_60_state_dict_best.pth
   ```

5. 発表に向けて整理する
   - 入力軌道，GT，予測軌道の違いを図で示す
   - どの対象が予測しやすく，どの対象が予測しにくいか説明する
   - 変更した条件と，その狙いを説明する
   - 評価値と可視化結果の両方を使って考察する

## レポートに含める内容

1. 課題1：可視化・理解
   - 可視化したファイル名（少なくとも3ファイル）
   - 入力軌道，GT，予測軌道を重ねた図
   - ボールと選手の動きの特徴
   - ボール・左チーム・右チームに分けた平均誤差
   - 最も予測が外れている箇所と，その理由の考察

2. 課題2：特徴量・入力条件の変更
   - 変更した内容（少なくとも2条件）
   - 変更前後の予測軌道の図
   - 評価値の比較
   - 改善または悪化した理由の考察

3. 課題3：モデル・条件の比較（発展）
   - 比較したモデルまたは設定
   - 学習条件
   - 評価結果
   - どちらが良かったか，その理由

## ヒント

- Pythonを使う場合
  - データ処理：`numpy`, `pandas`
  - 描画：`matplotlib`
  - 学習：`torch`
  - 進捗表示：`tqdm`
- 入力データ，GT，予測結果はCSV形式で扱われる
- ピッチの範囲は，ノートブック内ではおおよそ以下で描画されている

  ```text
  x: -52.5 から 52.5
  y: -34 から 34
  ```

- ノートブックの可視化では，入力の最後30フレーム，GT，予測結果を同じピッチ上に描画している
- 予測結果は通常以下に出力される

  ```text
  ./results/test/submission/
  ```

- 評価は以下のように実行する

  ```bash
  python example/evaluation.py --submit ./results/test/submission --gt ./test_old/gt --input ./test_old/input
  ```

- 対象別の平均誤差は，GTと予測の座標差から計算できる

  ```python
  error = ((pred_x - gt_x)**2 + (pred_y - gt_y)**2)**0.5
  mean_error = error.mean()
  ```

## 注意

- ノートブックは，`STP-challenge-2025` リポジトリ内で実行する前提になっている
- `--Sanity` を付けた場合は少数データでの動作確認用であり，本格的な性能比較には注意が必要である
- データセットを変更した場合は，ノートブック内のコメントに従い，必要に応じて `metadata.pkl` を削除すること
- 設定を変えて実行する場合は，前の予測結果を上書きしないように保存しておくこと
- グラフには凡例，軸ラベル，対象ファイル名を明記すること
- 評価値だけでなく，実際の軌道のずれを可視化して考察すること
- 発展課題までできなくても，まずは入力・GT・予測軌道の可視化と比較を完成させること
